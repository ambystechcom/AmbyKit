**Specify** turns a feature idea into a spec that captures **WHAT** and **WHY** — never technology.
It produces prioritized user stories (`US-#`), testable requirements in EARS notation (`FR-###`),
Given/When/Then acceptance criteria, and measurable success criteria (`SC-###`). Unknowns are flagged
inline as `[NEEDS CLARIFICATION: …]` rather than guessed.

## Command

```bash
/amby.specify <feature description>
```

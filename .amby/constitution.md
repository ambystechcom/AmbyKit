# Project Constitution — AmbyKit

> Durable, cross-feature principles that every AmbyKit phase must honor when building AmbyKit itself.
> Cite by number (e.g. "per Principle 1").

**Version:** 1.0.0  ·  **Last updated:** 2026-07-07

## Principle 1 — Author once, emit per tool
There is exactly one source of truth for the workflow: `src/templates/` and `src/prompts/`. Per-tool
files are always **generated** by emitters, never hand-authored as source. Any change to what the
assistant reads happens in the neutral source, followed by `ambykit sync`.

## Principle 2 — Subclass, don't duplicate
Shared behavior lives in `BaseEmitter` and `BaseCommand`. Adding a tool is a thin subclass that sets
its `commandSurface`/paths and overrides only what differs. Adding a CLI verb is a `BaseCommand`
subclass implementing `execute()`. Orchestration is never copy-pasted.

## Principle 3 — Token frugality is a feature
Prompts and templates minimize tokens the assistant reads/writes: progressive disclosure (load only
the artifact a phase needs via `@path`), reference by stable ID instead of restating, and patch in
place instead of regenerating. Machine reads (`dashboard`, `analyze`) run locally at zero model tokens.

## Principle 4 — Verify formats, never guess
Tool config formats come from `docs/tool-compatibility.md`, sourced from official docs. When a format
is uncertain, flag it and confirm against a live install — do not invent paths or frontmatter keys.

## Principle 5 — Testing standards
Every emitter has a snapshot test asserting its generated paths and frontmatter match the
compatibility matrix. Every functional requirement maps to a test. `npm run typecheck` and `npm test`
pass before commit.

## Principle 6 — Dogfood and stay self-hosted
AmbyKit is built with AmbyKit: features are specced under `specs/` using the workflow, and the repo's
own tool files are generated by `ambykit sync`. CI asserts `sync` produces no diff.

## Principle 7 — Least surprise & least privilege
Default to project-scoped, reversible changes. Never write user-level or out-of-tree files (e.g.
`~/.copilot/mcp-config.json`) without explicit consent, and keep them out of the sync check.
